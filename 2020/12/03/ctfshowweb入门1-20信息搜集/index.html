ctf show web 入门1-20信息搜集

发表于 2020-12-03 更新于 2021-03-10 分类于 ctfshow web入门

花199买的题目真香，入门是真的很入门(但我还是疯狂看hint才行)，这里总结一下我看到的知识点，也算不虚此行。

web 1

直接看网页源码

web 2

ctrl+U查看网页源码，或者是在url前加上view-source:来查看源码

web 3

抓包看响应头

web 4

访问robots.txt文件，这个文件里有开发者不希望搜索引擎访问的部分

web 5

有些网站会有index.phps文件，我也不知道哪些会有，总之是信息搜集的方法之一(phps就是php的源代码文件，通常用于提供给用户（访问者）直接通过Web浏览器查看php代码的内容。)

web 6

在url后面直接添加www.zip得到源码压缩包

web 7

考察git泄漏，解法是直接加上/git/index.php；

web 8

考察svn泄漏，直接在url后加上.svn; 造成SVN源代码漏洞的主要原因是管理员操作不规范。“在使用SVN管理本地代码过程中，会自动生成一个名为.svn的隐藏文件夹，其中包含重要的源代码信息。但一些网站管理员在发布代码时，不愿意使用‘导出’功能，而是直接复制代码文件夹到WEB服务器上，这就使.svn隐藏文件夹被暴露于外网环境，黑客可以借助其中包含的用于版本信息追踪的‘entries’文件，逐步摸清站点结构。”

web 9

提示使用vim时死机了，所以去看vim的备份文件，在url后加上index.php.swp

web 10

直接f12看cookie

web 11

DNS检查网站运维；TXT 记录，一般指为某个主机名或域名设置的说明。

web 12

ez。

web 13

document文件里有密码泄漏。

web 14

根据题目提示访问/editor,然后在文件上传处找到文件空间，var下的www目录是1部署的文件，找到flag位置，访问即可。

web 15

网页中的一些公开邮箱可能会泄漏信息，比如你所在的城市，进入/admin后点击忘记密码然后根据邮箱地址完成密码重置即可。

web 16

考察PHP探针php探针是用来探测空间、服务器运行状况和PHP信息用的，探针可以实时查看服务器硬盘资源、内存占用、网卡 流量、系统负载、服务器时间等信息。 url后缀名添加/tz.php 版本是雅黑PHP探针，然后查看phpinfo搜索flag。

web 17

要找真实ip，直接ping就完事儿了。

web 18

这种网页小游戏肯定是不可能去玩的啦，快进到看js文件然后解码。

web 19

题解告诉我前端有加密，所以post提交用户名和密码。

web 20

mdb文件是早期asp+access构架的数据库文件，直接查看url路径添加/db/db.mdb 下载文件通过txt打开或者通过EasyAccess.exe打开搜索flag

-------------结束啦!感谢阅读!-------------
