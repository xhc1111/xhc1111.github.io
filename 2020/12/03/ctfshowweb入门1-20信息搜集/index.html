ctf show web 入门1-20信息搜集

Posted on 2020-12-03 Edited on 2020-12-06

花199买的题目真香，入门是真的很入门(但我还是疯狂看hint才行)，这里总结一下我看到的知识点，也算不虚此行。

web 1 : 直接看网页源码

web 2 : ctrl+U查看网页源码，或者是在url前加上view-source:来查看源码

web 3 : 抓包看响应头

web 4 : 访问robots.txt文件，这个文件里有开发者不希望搜索引擎访问的部分

web 5 : 有些网站会有index.phps文件，我也不知道哪些会有，总之是信息搜集的方法之一(phps就是php的源代码文件，通常用于提供给用户（访问者）直接通过Web浏览器查看php代码的内容。)

web 6 : 在url后面直接添加www.zip得到源码压缩包

web 7 : 考察git泄漏，解法是直接加上/git/index.php；git泄漏

web 8 : 考察svn泄漏，直接在url后加上.svn; 造成SVN源代码漏洞的主要原因是管理员操作不规范。“在使用SVN管理本地代码过程中，会自动生成一个名为.svn的隐藏文件夹，其中包含重要的源代码信息。但一些网站管理员在发布代码时，不愿意使用‘导出’功能，而是直接复制代码文件夹到WEB服务器上，这就使.svn隐藏文件夹被暴露于外网环境，黑客可以借助其中包含的用于版本信息追踪的‘entries’文件，逐步摸清站点结构。”

web 9 : 提示使用vim时死机了，所以去看vim的备份文件，在url后加上index.php.swp

web 10 : 直接f12看cookie
